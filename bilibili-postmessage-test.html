B站嵌入播放器 postMessage 验证 v2

确认 B站 player.html 是否接收 postMessage 播放/暂停命令，并观察是否回传状态。
测试前：先点一下下方 iframe 内部任意位置激活它，再看画面是否在播（静音自动播放）。
关键观察：①点 pause 类按钮后，画面是否定格；②上方回传计数是否 > 0。

已收到回传：0 条
格式 A — {command:'pause'|'play'|'toggle'}（社区常用）
格式 B — {action:'play'|'pause'} / {method:'play'|'pause'}
格式 C — YouTube 风格 {event:'command',func:'playVideo'|'pauseVideo'}
格式 D — 直播 setPlayer- 字符串

判读：
• 若某格式让画面定格且回传计数 > 0 → 双向同步可行（路线 A，圆能真停）。
• 若画面定格但计数 = 0 → 命令有效但不回传，走乐观更新（圆停、B站大概率也停，但不 100% 确认）。
• 若画面不停且计数 = 0 → 此路在当前 B站版本不通，圆只能常转。
